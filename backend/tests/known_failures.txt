# GROUP 1 — non-admin roles get 404 on records they don't own (8 tests)
# ---------------------------------------------------------------------------
# Root cause: app/core/authorization.py:24 `authorize_owner()` — only ADMIN
# bypasses the ownership check, so a VIEWER / TEAM_MEMBER / ACCOUNTANT reading
# a contact created by someone else gets NotFoundError (404). The tests expect
# a viewer to be able to READ a contact they don't own.
#
# This is a product decision about the permission model, not a typo: fixing it
# widens data access. Someone needs to decide whether the model is
# "ownership gates reads too" (tests are wrong) or
# "ownership gates writes; org members can read" (authorization.py is wrong).
