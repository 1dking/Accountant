# RESOLVED — permission model (8 tests, was GROUP 1)
# ---------------------------------------------------------------------------
# Fixed. Root cause was app/core/authorization.py: only ADMIN bypassed the
# ownership check, so a VIEWER / TEAM_MEMBER / ACCOUNTANT got a 404 on any
# record they didn't personally create — which made the VIEWER role useless by
# construction and stopped team members from sharing a CRM.
#
# Ownership is no longer a second gate on shared business records; the route
# layer's require_role already decides what each role may do. Private resources
# (documents, meetings, SMTP configs) remain strictly owner-scoped, and CLIENT
# is deliberately NOT staff. See security/access_control.md and
# tests/integration/test_permission_model.py.
#
# A cashbook soft-delete bug (get_entry didn't hide deleted rows, so a deleted
# entry still returned 200) was fixed alongside it.
